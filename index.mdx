Stlite brings [Streamlit](https://streamlit.io/) to the browser by running Python entirely via [Pyodide](https://pyodide.org/) — a WebAssembly port of CPython. You can embed a full Streamlit app in a static web page, integrate it into a React app, bundle it as an Electron desktop application, or export it as a shareable URL — all without a Python server.

## How It Works

Stlite runs CPython inside your browser using [Pyodide](https://pyodide.org/), a WebAssembly build of Python. The Streamlit frontend runs as normal in the browser, but instead of talking to a Python server over a WebSocket, it communicates with a Pyodide kernel running in a [Web Worker](https://developer.mozilla.org/en-US/docs/Web/API/Web_Workers_API). This means:

- **No server required** — the entire app runs client-side
- **No installation for end-users** — just visit a URL
- **Pyodide packages** — install Python packages at runtime via `micropip`
- **Full Streamlit API** — most Streamlit widgets and features work as-is

## Try It Online

Visit [Stlite Sharing](https://edit.share.stlite.net/) to write and run a Streamlit app directly in your browser — no installation needed.

<Note>
  Stlite runs Python in the browser via WebAssembly. Some Python packages that rely on native C extensions may not be available. See the [Limitations guide](/guides/limitations) for details.
</Note>
